# AI Team Manager App Report

## Overview

AI Team Manager is a local Windows desktop command center for coordinating AI coding work across projects. It combines an Electron desktop shell, a Next.js/React interface, and a FastAPI backend backed by SQLite. The app is designed to let a user open a project folder, inspect its files, create implementation tasks, assign work to AI agents, monitor progress, and review the resulting changes before tasks are completed.

The central idea is that Codex acts as the lead coordinator while worker agents, especially Codex Worker, perform implementation tasks inside the selected workspace. The system keeps project state, task state, agent status, chat messages, review history, and captured file diffs in a local database.

## What The App Does

The application gives the user a command-center style workspace for managing software projects. From the desktop UI, the user can add or select a project, scan the real local project directory, browse files, open source files in a viewer, and inspect ongoing work in a multi-panel layout.

Users can create tasks manually or ask Codex to create tasks from natural language prompts. The backend routes implementation-style requests such as build, fix, add, write, or implement to Codex Worker. Direct commands to worker agents are intentionally blocked in the V1 workflow so that Codex remains the coordinator.

Once a task is assigned, the backend agent manager advances it through the workflow. Tasks move from assigned to working, then to review, and finally to completed after approval. The app records agent logs, task progress, review decisions, and file changes so the user can inspect what happened.

## Main Features

- Multi-project workspace management with local project paths and active branch tracking.
- Real directory scanning for selected project folders, with common build and dependency folders ignored.
- File browsing and source preview through the web interface.
- Task creation, assignment, progress tracking, and status grouping.
- Agent registry for Codex, Codex Worker, AntiGravity, and configurable custom agents.
- Live-style agent logs and status updates in the right panel and bottom inspector.
- Timeline chat where user prompts are saved and Codex responses are generated by the backend.
- Review workflow that prevents tasks from being marked complete without an approved Codex review.
- SQLite persistence for projects, tasks, agents, messages, reviews, and file changes.
- Electron folder picker integration for opening local project folders from the desktop app.

## Architecture

The project is organized as a small multi-app workspace:

- `apps/desktop` contains the Electron shell. It opens the Next.js app in a desktop window and exposes native features such as folder selection through a preload bridge.
- `apps/web` contains the Next.js and React user interface. It renders the workspace panels, project navigator, activity feed, task list, agent status panel, file viewer, and bottom inspector.
- `backend` contains the FastAPI server. It exposes REST endpoints for health checks, project management, file scanning, file reading, task updates, agent management, chat messages, reviews, and database reseeding.
- `shared/types` contains shared TypeScript types used by the frontend workspace.
- `scripts` contains PowerShell helpers for setup, backend startup, checks, tests, and smoke testing.

The backend initializes a local SQLite database and seeds default data. It runs a recurring agent simulation loop that checks for assigned and working tasks, updates agent state, captures diffs, and creates review records.

## Agent Workflow

Codex is the lead coordinator in the product workflow. When the user asks for implementation work, Codex creates a task and assigns it to Codex Worker. Codex Worker is intended to run real Codex implementation work in the selected project directory, leave changes as an uncommitted git diff, and submit those changes for review.

The `RealCodexAdapter` supports two execution paths:

- A local `openai-codex` Python SDK client when available.
- A fallback to the `codex exec` CLI when the SDK is not installed.

After the worker finishes, the adapter captures the current git diff, parses changed files, stores those changes in the database, and moves the task into review. Codex then performs a read-only review pass and records either approval or requested changes.

AntiGravity is also registered as a specialized backend-oriented worker using a CLI adapter. Other agents can be represented with mock adapters for simulated progress.

## Data Storage

The app stores its state in SQLite rather than relying only on frontend mock data. The database schema includes tables for:

- Projects
- Agents
- Tasks
- Messages
- Reviews
- File changes
- Schema metadata

This allows the desktop app, backend, and frontend polling loop to share a durable local view of project activity.

## How To Run It

The root `package.json` defines scripts for setup, development, production-style startup, type checking, building, and tests.

Typical setup:

```powershell
npm run setup
```

Development startup:

```powershell
npm run dev
```

This starts the FastAPI backend, the Next.js web app, and the Electron desktop shell together.

Testing:

```powershell
npm run test
```

## Summary

AI Team Manager is an early but functional local orchestration tool for AI-assisted software development. It is not just a chat UI: it combines project browsing, task dispatch, agent monitoring, database-backed workflow state, git diff capture, and Codex review into one desktop workspace. Its main purpose is to make AI coding work more structured, visible, and reviewable before changes are considered complete.

## Intended User

This app is intended for developers who want a local command center for managing AI-assisted coding work across one or more projects. Instead of sending prompts to separate agents in isolation, the user can keep projects, tasks, agent activity, file inspection, implementation diffs, and review decisions in a single desktop workflow.
